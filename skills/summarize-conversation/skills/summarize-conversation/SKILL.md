# 会話まとめスキル

直近の会話でやりとりした内容をマークダウンファイルにまとめて保存する。

## 実行手順

### 1. 会話内容の分析

直近の会話から以下を抽出:

- **調査テーマ**: 何について調査・議論したか
- **主要な発見**: 重要なポイントや結論
- **コード例**: 議論されたコードスニペット（あれば）
- **参考リンク**: 共有された URL やリソース
- **次のステップ**: 今後のアクション項目（あれば）

### 2. ファイル名の決定

調査内容を端的に表す日本語ファイル名を生成:

- 形式: `{調査概要}.md`
- 例: `React Server Components の調査.md`
- 例: `GraphQL 認証実装の検討.md`
- 例: `パフォーマンス改善の分析.md`

ファイル名は簡潔に（20文字以内推奨）、内容が分かるようにする。

### 3. 保存先ディレクトリの準備

保存先: 作業リポジトリの `_docs` ディレクトリ

ディレクトリが存在しない場合は作成:

```bash
mkdir -p _docs
```

### 4. マークダウンファイルの作成

以下のフォーマットでファイルを作成:

```markdown
# {タイトル}

> 作成日: {YYYY-MM-DD}

## 概要

{調査の背景や目的を1-2文で}

## 調査内容

### {セクション1}

{詳細内容}

### {セクション2}

{詳細内容}

## 結論・まとめ

- {主要な結論1}
- {主要な結論2}
- {主要な結論3}

## 参考リンク

- [{リンクタイトル}]({URL})

## 次のステップ

- [ ] {アクション1}
- [ ] {アクション2}
```

### 5. ファイルの保存

Write ツールを使用して `_docs/{ファイル名}.md` に保存し、保存したファイルパスをユーザーに報告。

例: `_docs/React Server Components の調査.md`

## 注意事項

- 会話内容を忠実にまとめる（勝手に情報を追加しない）
- コードブロックは適切な言語タグを付ける
- 機密情報や秘密鍵などが含まれていないか確認
- 長すぎる内容は適切に要約する
